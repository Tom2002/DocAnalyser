SearchInputCtrl
controller in module docanalyser

Description

Controller for the management of the search bar. Here user interactions are treated and the cooccs hash is maintained.

Methods

  • change()

    The user manually change the search bar input. Synchronize the search bar input with the status of the checkboxes.

  • chooseRelevantTerm()

    The user has selected a relevant term. Appends the term to the existing search bar input.

  • chooseSuggestion()

    The user has selected a search suggestion. Appends the suggestion to the existing search bar input.

  • click()

    The user has clicked the search button. The url is updated with the new search terms.

  • init()

    Initialization stuff. Load the stored cooccs hash.

  • locationChangeSuccess()

    The url has changed -> maybe back or previous button was pressed. Synchronize the search bar input with the url. Disable the browser back button for the chronicle view.

  • searching()

    The user has entered something in the search bar. Search suggestions and relevant terms are determined here.

  • selectedTermsChanged()

    A term was selected or deselected. The term is added or removed from the search bar.

  • updateCooccs()

    The user has performed a google search. Here the cooccs hash is updated. Only nouns and names are considered.

  • watchGroup()

    Terms were selected or deselected. Maintain the selectedTerms and searchBarArr properties.

Properties

  • searchBar:Object

    The search bar Object.

  • searchBarArr:Array

    The search bar input as array.

  • cooccs:Object

    The cooccs hash.

  • selectedTerms:Array

    Contains the selected Terms (checkboxes).

  • showSuggestions:boolean

    Determines whether search suggestions are displayed.

  • autoComplete:boolean

    Determines whether search suggestions and relevant terms are displayed.

  • searchSuggestions:Array

    Contains the search suggestions.

  • relevantTerms:Array

    Contains the relevant terms.
