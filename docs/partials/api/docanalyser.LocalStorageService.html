LocalStorageService
service in module docanalyser

Description

This service provides a custom interface to the Webstorage API. Data can be stored persistently in the user's local memory. The standard way to store a object is: addObject(myObject); or setObject(object); and then saveObject(object); The standard way to load a object is: loadObject(); and then getObject();

Methods

  • clearQueries()

    remove all Query Items stored in the local Storage.

    Returns
    boolean

    indicates whether the deletion was successful

    Example

    ret = LocalStorageService.clearQueries();

  • clearQueries(type)

    detects whether localStorage is both supported and available Credits to: https://developer.mozilla.org/de/docs/Web/API/Web_Storage_API/Using_the_Web_Storage_API

    Parameters
    ParamTypeDetails
    typeString

    specifies which storage type should be checked

    Returns
    boolean

    indicates whether the storage is available

    Example

    ret = LocalStorageService.storageAvailable("localStorage");

Properties

  • queries:Array

    Global Array that contains the stored queries.

  • chronicleStatus:String

    Global String that indicates whether the user wants to save queries.

  • cooccs:Object

    Global Object that contains the stored cooccs hash.
